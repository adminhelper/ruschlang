# 루슐랭 실행 방법

## 1) 사전 준비

- Node.js 20+ 또는 Bun
- Docker 또는 로컬 MySQL
- `git` 체크아웃 후 프로젝트 루트로 이동

## 2) DB 설정

```bash
cp server/.env.example server/.env
# MYSQL_* 값과 ADMIN_TOKEN, NAVER_MAP_CLIENT_ID 갱신
```

`db/mysql/schema.sql`을 사용해 기본 테이블을 생성합니다.

```bash
mysql -h 127.0.0.1 -P 3307 -u ruschlang -pruschlang ruschlang < db/mysql/schema.sql
```

로드맵/게시글/댓글 테이블을 새로 반영하려면 아래를 실행합니다.

```bash
mysql -h 127.0.0.1 -P 3307 -u ruschlang -pruschlang ruschlang < docs/db/sql/roadmaps.sql
mysql -h 127.0.0.1 -P 3307 -u ruschlang -pruschlang ruschlang < docs/db/sql/posts.sql
mysql -h 127.0.0.1 -P 3307 -u ruschlang -pruschlang ruschlang < docs/db/sql/post_comments.sql
```

## 3) 서버 실행

```bash
cd server
bun install
bun run dev
```

기본 주소: `http://localhost:3001`

## 4) 클라이언트 실행

루트에서 정적 서버를 띄우거나, 브라우저에서 `index.html`을 열어도 동작합니다.

```bash
# 예: static-server
npx serve .
# 또는
bunx serve .
```

서버 API 기본 주소가 변경된 경우 `app.js` 상단의 `API_BASE_URL`을 맞춰 주세요.

## 5) 점검 체크리스트

- 비회원
  - 지도 / 탐방 게시판 탭만 보이는지 확인
  - 맛집 등록 / 로드맵 작성 폼이 숨겨지는지 확인
- 회원 로그인
  - 탐방지도, 탐방게시판, 맛집 로드맵 탭만 보이는지 확인
  - 로드맵 등록 가능
- 관리자 로그인
  - 맛집 등록 탭이 보이고 등록이 가능한지 확인
  - 관리자 코드 기반 인증이 유지되는지 확인
- 맛집 지도
  - `Naver Map JS SDK` 인증 실패 시 확인 항목
    - `server/.env`의 `NAVER_MAP_CLIENT_ID` 값이 실제 발급 키와 일치하는지
    - 서비스 URL이 오픈API 앱 설정의 허용 URL/도메인에 등록되었는지
    - 지도 키를 수동 입력할 때는 공백/줄바꿈 없이 정확한 키 문자열인지
  - `server/.env`의 `NAVER_MAP_CLIENT_ID`로 자동 연동되는지
  - 없으면 입력한 키로 수동 연동되는지

## 6) API 정리

- `GET /api/health`
- `GET /api/map/config`
- `GET /api/map/sdk/naver.js`
- `GET /api/admin/ping`
- `GET /api/restaurants` (`sort=latest|name|rating`)
- `POST /api/restaurants` (관리자 토큰 필요)
- `DELETE /api/restaurants/:id` (관리자 토큰 필요)
- `POST /api/restaurants/:id/reviews` (회원/비회원 가능)
- `GET /api/roadmaps`
- `POST /api/roadmaps`
- `DELETE /api/roadmaps/:id` (관리자 토큰 필요)
- `GET /api/posts` (비회원: 승인글만, 회원: 승인+대기, 관리자: 전체)
- `POST /api/posts` (회원/관리자)
- `PUT /api/posts/:id` (관리자)
- `PATCH /api/posts/:id/approve` (관리자, status: pending/approved/rejected)
- `DELETE /api/posts/:id` (관리자)
- `GET /api/posts/:id/comments`
- `POST /api/posts/:id/comments` (회원/관리자)

## 7) 권한 점검 체크

- 비회원
  - 지도/탐방 게시판은 열람 가능
  - 로그인 버튼이 보이고 게시글 쓰기/댓글은 비활성
- 회원
  - 게시글 작성 가능(등록 시 승인 대기)
  - 로드맵 작성 및 댓글/대댓글 작성 가능
 - 관리자
   - 맛집 등록 탭 노출 및 등록
   - 게시글 승인/반려/수정/삭제 가능

## 8) 테스트 실행

- Playwright 테스트(저장된 API 응답을 모킹):

```bash
bun run test
```

- 빠른 수동 스모크 테스트:

```bash
bun run smoke
```
